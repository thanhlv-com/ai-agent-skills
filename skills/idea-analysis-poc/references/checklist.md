# Idea Analysis POC Checklist

## Input Framing
- Restate idea in one clear sentence.
- Identify target user/persona and primary pain point.
- Capture constraints: timeline, team capacity, budget, compliance, platform.
- Mark assumptions vs confirmed facts.

## Problem and Opportunity
- Define measurable problem statement.
- Estimate impact if solved (business or user value).
- List current alternatives and their gaps.

## Option Expansion
- Propose 2 to 3 solution approaches.
- Compare each option by:
  - delivery speed,
  - implementation complexity,
  - technical risk,
  - operational cost,
  - scalability potential.
- Choose recommended option with explicit rationale.

## POC Artifact Pack
- POC brief: objective, scope, non-goals, hypotheses.
- Solution outline: architecture, key components, core flow, interfaces.
- Execution plan: milestones, owners, dependencies, timeline.
- Validation plan: success metrics, test scenarios, experiment design.
- Risk register: top risks, triggers, mitigations, fallback.

## Quality Gate
- Artifacts are concrete enough for immediate kickoff.
- Metrics are measurable and time-bound.
- Risks include mitigation and decision points.
- Open questions are clearly assigned for follow-up.
